Documentation.

Tout ce qu'il faut pour intégrer HenergyqueAI à votre workflow — en ligne de commande, par API ou directement dans votre IDE.

Démarrage rapide

Installez le client officiel, puis envoyez votre première requête en moins d'une minute.

shell
pip install henergyque
python
from henergyque import Client

client = Client()
reponse = client.chat("Écris un tri fusion en Python")
print(reponse.texte)

Authentification

Chaque requête est authentifiée par une clé API, disponible dans votre espace personnel. Exposez-la via une variable d'environnement — jamais en dur dans le code.

shell
export HENERGYQUE_API_KEY="hq-…"
La clé est liée à votre plan : les quotas du plan Gratuit s'appliquent automatiquement tant que vous n'êtes pas passé en Pro.

SDKs officiels

Deux SDKs maintenus par Henergyque : Python (voir Démarrage rapide) et JavaScript / TypeScript. Les deux couvrent chat, streaming et gestion des erreurs.

shell
npm install @henergyque/sdk
typescript
import { Client } from "@henergyque/sdk";

const client = new Client();
const reponse = await client.chat("Écris un debounce en TypeScript");
console.log(reponse.texte);

Envoyer une requête

L'API expose un unique endpoint de chat. Envoyez l'historique des messages, recevez la réponse du modèle.

curl
curl https://api.henergyque.ai/v1/chat \
  -H "Authorization: Bearer $HENERGYQUE_API_KEY" \
  -H "Content-Type: application/json" \
  -d '{
    "messages": [
      { "role": "user", "content": "Écris un debounce en TypeScript" }
    ],
    "max_tokens": 1024
  }'
réponse · json
{
  "role": "assistant",
  "content": "```typescript\nfunction debounce…",
  "usage": { "input_tokens": 18, "output_tokens": 142 }
}

Paramètres

Référence des champs acceptés par /v1/chat.

Champ
Type
Description
messages
array · requis
Historique de la conversation, objets { role, content }.
system
string
Instructions globales : conventions, langage cible, style de réponse.
max_tokens
int · défaut 1024
Longueur maximale de la réponse, dans la limite du plan.
temperature
float · défaut 0.2
0 = déterministe, 1 = créatif. Restez bas pour du code.
stream
bool · défaut false
Réponse en flux SSE — voir la section Streaming.

Contexte de code

Joignez des fichiers à la requête pour que le modèle raisonne sur votre code réel — indispensable pour le debug et le refactoring.

python
reponse = client.chat(
    "Corrige le bug de pagination",
    fichiers=["src/api.py", "src/pagination.py"]
)
Le contenu des fichiers compte dans les tokens de la requête. L'extension IDE sélectionne automatiquement les fichiers pertinents du projet ouvert.

Streaming

Ajoutez "stream": true pour recevoir la réponse token par token (Server-Sent Events) — idéal pour afficher le code au fil de la génération.

python
for morceau in client.chat("Écris un tri fusion", stream=True):
    print(morceau.texte, end="", flush=True)
flux · sse
event: delta
data: { "texte": "function " }

event: delta
data: { "texte": "debounce" }

event: fin
data: { "usage": { "output_tokens": 142 } }

Extension IDE

Disponible pour VS Code et JetBrains (plan Pro). Complétion en ligne, chat contextuel et refactoring directement dans l'éditeur.

vs code
code --install-extension henergyque.henergyque-ai

Codes d'erreur

L'API utilise les codes HTTP standard. Le corps de la réponse contient toujours un champ "erreur" détaillé.

Code
Signification
401
Clé API absente ou invalide. Vérifiez HENERGYQUE_API_KEY.
429
Quota atteint (plan Gratuit) ou trop de requêtes simultanées. Réessayez avec un backoff exponentiel.
422
Requête mal formée — champ manquant ou max_tokens au-delà de la limite du plan.
500
Erreur interne. Consultez la page Statut et réessayez.

Bonnes pratiques

Soyez précis sur le résultat attendu

« Écris un debounce en TypeScript, typé, sans dépendance » donne un meilleur résultat que « fais un debounce ».

Donnez le contexte du projet

Joignez les fichiers concernés ou décrivez la stack dans system : framework, version, conventions.

Itérez en conversation

Gardez l'historique dans messages et affinez : « ajoute la gestion d'erreur », « convertis en async ».

Demandez les tests

Terminez par « écris aussi les tests unitaires » — le modèle est entraîné à produire des tests cohérents avec son propre code.

Sécurité & confidentialité

Toutes les requêtes transitent en TLS 1.3. Votre code n'est jamais utilisé pour entraîner les modèles partagés, et les requêtes ne sont pas conservées au-delà du traitement.

Les données de fine-tuning restent isolées dans votre espace et peuvent être supprimées à tout moment. Pour une confidentialité totale, le plan API permet un déploiement on-premise : rien ne quitte votre infrastructure.

Révoquez ou régénérez vos clés API à tout moment depuis votre compte. Une clé compromise ? Contactez-nous.
